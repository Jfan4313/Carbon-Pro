# 安全说明 (Security)

本文档记录零碳项目收益评估软件的安全相关信息和已知漏洞。

---

## 已知安全漏洞

### 1. xlsx 包 (SheetJS) - 高危

**漏洞 ID**: GHSA-4r6h-8v6p-xvw6, GHSA-5pgg-2g8v-p4x9
**严重性**: High
**状态**: 等待上游修复

**描述**:
- 原型污染漏洞 (Prototype Pollution)
- 正则表达式拒绝服务攻击 (ReDoS)

**影响**: 使用 Excel 导出功能可能受影响

**缓解措施**:
- 仅从可信来源导入 Excel 文件
- 限制上传文件大小
- 等待 SheetJS 团队发布安全修复

**跟踪**: https://github.com/advisories/GHSA-4r6h-8v6p-xvw6

---

### 2. yauzl 包 - 中危

**漏洞 ID**: GHSA-gmq8-994r-jv83
**严重性**: Moderate
**状态**: 等待 Electron 更新

**描述**: off-by-one 错误，可能导致解压时出现问题

**影响**: Electron 应用解压操作可能受影响

**缓解措施**:
- 仅从可信来源下载安装包
- 验证下载文件的哈希值

**跟踪**: https://github.com/advisories/GHSA-gmq8-994r-jv83

## 安全最佳实践

### 开发环境

- 不要在代码中硬编码敏感信息（API 密钥、密码等）
- 使用环境变量存储敏感配置
- 定期更新依赖包

### 生产部署

- 使用 HTTPS 部署
- 配置 CORS 限制访问域名
- 定期检查安全漏洞 (`npm audit`)
- 启用安全 HTTP 头（已在 vercel.json 中配置）

### 用户数据

- 使用 Supabase 进行认证和数据存储（已加密）
- 不在本地存储敏感信息
- 使用 IndexedDB 进行本地数据持久化

## 报告安全问题

如果你发现安全漏洞，请：

1. 不要公开 Issue
2. 发送邮件至项目维护者
3. 详细描述漏洞和复现步骤
4. 等待确认后再公开披露

我们会：
- 在 48 小时内确认收到
- 在 7 天内评估并响应
- 协调发布时间表
- 给予安全研究员适当的荣誉

## 依赖包安全

### 定期检查

建议每月运行一次安全检查：

```bash
npm audit
```

### 自动修复

对于有可用修复的漏洞：

```bash
npm audit fix
```

**注意**: 使用 `--force` 选项可能导致破坏性更改，请谨慎使用。
